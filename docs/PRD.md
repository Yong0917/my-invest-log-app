# my-invest-log-app MVP PRD

## 핵심 정보

**목적**: 로그인한 사용자가 보유 주식의 현재가와 수익률을 한눈에 확인할 수 있는 개인 포트폴리오 대시보드 제공
**사용자**: 국내외 주식을 직접 투자하며 수익률을 빠르게 파악하고 싶은 개인 투자자

---

## 사용자 여정

```
1. 서비스 진입
   ↓ 비로그인 상태로 접근

2. 로그인 페이지 (/auth/login)
   ↓ 이메일/비밀번호 입력 및 로그인 버튼 클릭

3. 로그인 성공
   ↓ 자동 리디렉션 → 대시보드 페이지

4. 대시보드 페이지 (/protected/dashboard)
   ↓ 보유 주식 존재 여부 분기

   [보유 주식 없음] → "첫 주식 등록하기" CTA 표시
                     ↓ 버튼 클릭
                     → 주식 등록 모달 오픈

   [보유 주식 있음] → 요약 카드 + 종목 테이블 표시
                     ↓ "주식 추가" 버튼 클릭
                     → 주식 등록 모달 오픈

5. 주식 등록 모달
   ↓ 종목코드 입력 → 검색 → 종목명 자동 조회 확인
   ↓ 보유 수량 + 평균 매수가 입력 → 저장 버튼 클릭

6. 저장 완료
   ↓ 모달 닫힘 → 대시보드 자동 갱신
   → Yahoo Finance API로 현재가 실시간 조회
   → 수익률 자동 계산 및 테이블 갱신

7. 종목 수정/삭제 (대시보드에서)
   ↓ 종목 행의 수정 또는 삭제 버튼 클릭

   [수정] → 수정 모달 오픈 → 수량/매수가 변경 → 저장
   [삭제] → 삭제 확인 → 목록에서 제거
```

### portfolios (보유 주식 정보)

| 필드 | 설명 | 타입/관계 |

> **유니크 제약**: (user_id, ticker) 조합 중복 불가

## 수익률 계산 공식

```
평가금액   = 현재가 × 보유수량
투자금액   = 평균매수가 × 보유수량
평가손익   = 평가금액 - 투자금액
수익률(%) = (평가금액 - 투자금액) / 투자금액 × 100

포트폴리오 총 평가금액 = Σ 개별 평가금액
포트폴리오 총 투자금액 = Σ 개별 투자금액
포트폴리오 전체 수익률 = (총 평가금액 - 총 투자금액) / 총 투자금액 × 100
```

## 개발 단계

MVP는 백엔드 연동 전에 화면과 인터랙션을 먼저 검증하는 **3단계 접근 전략**을 따릅니다.

### Phase 0 — UI 프로토타입

> **목적**: 실제 기능 없이 화면 구조와 사용자 플로우를 빠르게 검증

| 항목 | 내용 |
|------|------|
| **개발 범위** | 정적 마크업 + 스타일링만 구현 (API 호출, 상태 관리 없음) |
| **반응형** | Mobile First 기반, 모바일 → 태블릿 → 데스크톱 순으로 설계 |
| **구현 대상** | 로그인 페이지, 대시보드 레이아웃, 주식 등록/수정 모달 UI |
| **검증 목표** | 사용자 플로우가 직관적인지, 레이아웃이 정보를 잘 전달하는지 확인 |
| **완료 기준** | 모든 페이지/모달이 정적으로 렌더링되고, 모바일/데스크톱에서 레이아웃이 깨지지 않음 |

### Phase 1 — 더미 데이터 기반 개발

> **목적**: Mock 데이터로 화면 동작과 UX를 사전 검증한 뒤 실제 API 연동 준비

| 항목 | 내용 |
|------|------|
| **개발 범위** | 상태 관리 + 인터랙션 구현 (Supabase, Yahoo Finance API 연결 없음) |
| **Mock 데이터** | 하드코딩된 포트폴리오 데이터로 테이블, 요약 카드, 수익률 계산 동작 구현 |
| **구현 대상** | 주식 추가/수정/삭제 인터랙션, 수익률 실시간 계산, 빈 상태(Empty State) 처리 |
| **검증 목표** | 수익률 계산 로직 정확성 확인, 모달 열림/닫힘 등 UX 흐름 검증 |
| **완료 기준** | Mock 데이터 기준으로 모든 CRUD 인터랙션이 오류 없이 동작하고 수익률이 올바르게 계산됨 |

### Phase 2 — 실제 기능 연동 (MVP 완성)

> **목적**: Phase 0~1에서 검증된 UI/UX 위에 실제 백엔드와 API를 연결

| 항목 | 내용 |
|------|------|
| **개발 범위** | Supabase DB 연동, Yahoo Finance API 연동, Supabase Auth 연동 |
| **구현 순서** | DB 마이그레이션 → 인증 → 포트폴리오 CRUD → 현재가 조회 → 수익률 계산 통합 |
| **완료 기준** | 실제 데이터로 전체 사용자 여정이 오류 없이 동작함 |
